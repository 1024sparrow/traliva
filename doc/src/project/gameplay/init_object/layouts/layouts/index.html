layouts

Пусть функция get_layout в одних случаях возвращает '$smartphone', а в других - '$desktop'. Тогда вот пример того, что может быть в свойстве layouts:

... $layouts: { $smartphone:'$catalogue', $desktop:{ $type: $TralivaKit.$Strip, $orient: #e#$TralivaKit__Strip__orient:v##, $items:[ { $_widget: '$adPanel', $size: '200px' }, '$catalogue' ] } }, ...

Рассмотрим, что имеем.

В случае, когда get_layout возвращает строку '$smatrphone', соответствующее поле в layouts имеет тип Строка (а не Объект). Значит, на весь экран будет растянут виджет из widgets, который хранится в свойстве '$catalogue'.

В случае, когда get_layout возвращает строку '$desktop', соответствующее поле в layouts имеет тип Объект. Этот объект должен иметь свойство $type(конструктор виджета). Значит, на весь экран будет растянут контейнер виджетов, созданный конструктором класса, указанным в свойстве $type. В документации по компоненту $TralivaKit.$Strip можно найти, что этот компонент параметризуется свойствами $orient и $items: перечисление типа $TralivaKit__Strip__orient и массив объектов {$_widget: ..., <опции дочернего элемента, см. справку по компоненту>}, описывающих дочерние виджеты, соответственно.

Свойство $items зарегистрировано в компоненте $TralivaKit.$Strip как свойство, описывающее набор дочерних виджетов. Это массив идентификаторов виджетов из состава widgets или объектов, описывающих дочерние виджеты (с обязательным свойством $_widget и свойствами, отображающими особенности размещения этого виджета в составе родительского виджета-контейнера).

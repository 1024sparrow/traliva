init

Здесь реализована функция Traliva.init(), а также здесь определены следующие классы, не имеющие смысла за пределами функции Traliva.init(): WidgetStateSubscriber и LogicsStateSubscriber.

WidgetStateSubscriber

Данный класс наследуется от класса StateSubscriber.

Является базовым для классов для описания виджета (возможно, контейнера для каких-то априорно неизвестных дочерних виджетов). Конструктор пользовательского класса, отнаследованного от данного класса, используется в свойстве widgets объекта, передаваемого параметром в функцию Traliva.init().

В том случае, когда пользовательский класс описывает контейнер виджетов, в пользовательском классе должен быть определён метод (статический) widgetsFields (без параметров), возвращающий список свойств, указанных в объекте, описывающем элемент лэйаута, по которым будут генерироваться виджеты, передаваемые третим параметром в конструктор при создании экземпляра данного класса.

Принимает три параметра:

  • p_wContainer - экземпляр класса Widget - в который в конструкторе пользовательского класса должно устанавливаться содержимое (метод setContent() класса Widget).
  • p_options - набор опций (options), который задал пользователь в объекте, передаваемом в Trliva.init(), в свойстве объекта widgets, соответствующего пользовательскому виджету или пользовательскому контейнеру.
  • p_widgets - используется только для элементов типа container - объект, в котором свойства такие, как указано в результате выполнения статического метода widgetsFields, а значения этих свойств - соответвующие виджеты.

При наследовании программист (которого мы обычно называем польозователем..) должен не забыть сделать следующие вещи:

  • установить содержимое виджету, передаваемому первым параметром в конструктор.
  • Если пользовательский класс представляет контейнер, то необходимо встроить дочерние виджеты(третий параметр) в тот виджет, который устанавливается в объект, передаваемый первым параметром
  • Возможно, программист должен обеспечить изменение размеров и позиций дочерних виджетов при изменении размеров виджета, передаваемого первым параметром

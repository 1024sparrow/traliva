НАВЕРХ

Изменения относительно предыдущей версии спецификации

  • Появились требования к структуре репозитория проекта - теперь должен остаться на все случаи жизни один репозиторий traliva вместо старых traliva, traliva_kit и traliva_starter. Появились требования в фреймворку, чтоб там был свой механизм сборки пользователького проекта под разные платформы.
  • Traliva.StateSubscriber: теперь в конструктор можно передать названия свойств(и, если надо, функции-обработчики), которые будут автоматически удаляться после первого оповещения подписчиков.
  • Новый тип элементов лэйаута - container (контейнер, определяемый пользователем) ; описан способ задания элемента лэйаута const_widget (раньше было указано, что такой есть, но как его отличить от widget, указано не было)
  • WidgetStateSubscriber - теперь может применяться не только для определения элементов лэйаута widget и const_widget, но и для определения пользовательских контейнеров (тип лэйаута container). Поэтому принимает дополнительный параметр - объект с дочерними виджетами (массивами дочерних виджетов), ассоциированными по ключам, переданным в описании лэйаута.

Формат описания приложения

В данной директории предполагается наличие двух диреторий (res и children) и трёх файлов (style.css, gameplay.js и project). Если вам не удобно всё писать в одном файле, вы можете разбить его на части, но обеспечить сборку вашего проекта в вышеуказанные две директории и три файла.

Итак, в результате компиляции (или просто сразу так) в данной директории должны находиться следующие файлы:

  • res/ - ресурсы, такие как картинки
  • children/ - подгружаемые части приложения (если, конечно, такие у вас есть)
  • gameplay.js - код корневой части приложения (если есть подгружаемые части, то они описываются отдельно)
  • style.css - файл стилей проекта
  • project - файл проекта

Здесь описаны классы, определённые внутри фреймворка Тралива.

Весь код пишется на ECMAScript 5 (а не на последней версии - (ECMAScript 6)) с некоторыми дополнительно поддерживаемыми конструкциями. Использование ECMAScript 5 связано с тем, что последняя версия ECMAScript поддерживается не всеми браузерами, поддерживается не в полном объёме, из-за чего использование ECMAScript 6 требует использование полифилов, что, по мнению автора, является неоправданным усложнением клиентского кода.

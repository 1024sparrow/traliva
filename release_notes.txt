Комментарии к релизу:

Комментарий 1.
==============
В спецификации этого не указано - при подключении фрэймворка в html-файле необходимо кроме того как подключить скрипт traliva.js, необходимо также подключить файл стилей
В указанном ниже примере предполагается, что в корне вашего проекта находится симолическая ссылка (или копия) на директорию compiled, находящуся в директории traliva, под именем traliva.

<!doctype html>
<html>
    <head>
        <meta name="viewport" content="width=device-width; user-scalable=0;" />
        <link href="traliva/_style/style.css" rel="stylesheet"/>
        <script src="traliva/traliva.js"></script>
        ...
    </head>
    <body>
        ...
        <noscript> Для работы сайта необходимо, чтобы поддержка JavaScript не была отключена.</noscript>
        <script src="gameplay.js"></script>
    </body>
</html> 

Комментарий 2.
===============
В спецификации указано, что объект, принимаемый параметром функцией Traliva.init(), может содержать свойство extender. Хоть в спецификации и не описан формат этого поля, данная версия фрэймворка это поле не поддерживает - издатель не может гаранитировать корректную реакцию на наличие такого поля, и от использования этого поля следует воздержаться до, как минимум, выхода следующей версии спецификации (1 марта 2019 года), где это поле будет описано.
.

Комментарий 3.
==============
Данная версия фрэймворка Тралива соответствует спецификации версии 1, находящейся по адресам github.com/1024sparrow/traliva_doc и traliva.ru
Фрэймворк не содержит готовых виджетов - здесь описаны только механизм создания и подключения модулей, их взаимодействия.
При разработке вам также может пригодиться библиотека готовых виджетов. Она находится по адресу github.com/1024sparrow/traliva_kit .
